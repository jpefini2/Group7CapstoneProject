Set Avaliability:

Context: The advisor is in the avalibaility page

Primary Flow:
1. The user selects the desired day.
2. The user selects/enters times they are avaliable for that day.
3. The user selects confirm. 
4. The system persists the information to the database.
5. The system prompts the user of successful update.

Alternate Flow:
	A4. The system fails to persist the information.
	A5. The system prompts the user of the error.
	A6. Return to primary flow 2.
